# 다음카페 Re: 글 수집기

Daum Cafe 게시판에서 사용자가 직접 볼 수 있는 글 중 제목이 `Re:`로 시작하는 글만 열람하고, 제목과 본문만 워드파일로 저장하는 Python Selenium 프로그램입니다.

대상 게시판:

https://cafe.daum.net/school-cont/eri

## 중요한 주의사항

- 이 프로그램은 로그인 자동화, 캡차 우회, 권한 우회 기능을 포함하지 않습니다.
- 사용자가 직접 로그인하고 게시판 접근 권한이 있는 상태에서만 수집합니다.
- 비공개 글이나 권한이 없는 글은 수집하지 않습니다.
- 로그인 정보, 아이디, 비밀번호는 코드에 저장하지 않습니다.
- 서버 부담을 줄이기 위해 글 하나를 열 때마다 2~5초 랜덤 대기합니다.
- Daum Cafe 화면 구조가 바뀌면 추출이 실패할 수 있습니다. 실패 시 `debug` 폴더에 스크린샷과 HTML이 저장됩니다.

## 파일 구성

- `main.py`: Selenium으로 게시판을 순회하고 워드파일을 만드는 실행 파일
- `gui.py`: 버튼과 입력칸이 있는 Windows GUI 실행 파일
- `requirements.txt`: 필요한 Python 패키지 목록
- `README.md`: 설치 및 실행 안내

## 설치 방법 (Windows 기준)

### 1. Python 설치 확인

Windows PowerShell 또는 명령 프롬프트에서 아래 명령을 실행합니다.

```powershell
python --version
```

Python 3.10 이상을 권장합니다. Python이 없다면 https://www.python.org/downloads/ 에서 설치하세요.

설치할 때 **Add python.exe to PATH** 옵션을 체크하는 것이 좋습니다.

### 2. Chrome 설치 확인

이 프로그램은 Google Chrome 브라우저를 사용합니다. Chrome이 설치되어 있어야 합니다.

ChromeDriver는 `webdriver-manager`가 자동으로 내려받습니다.

### 3. 가상환경 만들기

프로젝트 폴더에서 PowerShell을 열고 아래 명령을 실행합니다.

```powershell
python -m venv .venv
```

가상환경을 활성화합니다.

```powershell
.\.venv\Scripts\Activate.ps1
```

PowerShell 실행 정책 문제로 활성화가 막히면 아래 명령을 한 번 실행한 뒤 다시 활성화합니다.

```powershell
Set-ExecutionPolicy -Scope CurrentUser RemoteSigned
```

### 4. 패키지 설치

```powershell
pip install -r requirements.txt
```

## GUI 실행 방법

콘솔 입력이 불편하면 `run_gui.bat` 파일을 더블클릭해서 창 형태의 프로그램을 실행하세요.

PowerShell에서 실행할 때는 아래 명령을 사용할 수도 있습니다.

```powershell
.\.venv\Scripts\python.exe gui.py
```

GUI 창이 열리지 않으면 `run_gui_debug.bat`를 더블클릭하세요. 오류가 나면 창이 바로 닫히지 않고 내용을 보여주며, 같은 폴더에 `gui_error.log` 파일도 남깁니다.

GUI 사용 순서:

1. `브라우저 열기`를 누릅니다.
2. 열린 Chrome 창에서 직접 Daum에 로그인합니다.
3. 대상 게시판 목록이 보이는 상태인지 확인합니다.
4. GUI 창에서 시작 페이지와 종료 페이지를 입력합니다.
5. 저장 파일 위치를 확인하거나 `찾기`로 변경합니다.
6. `수집 시작`을 누릅니다.

시작 페이지와 종료 페이지를 비워두면 현재 페이지부터 감지된 마지막 페이지까지 수집합니다.

## 콘솔 실행 방법

```powershell
python main.py
```

실행하면 Chrome 브라우저가 열리고 대상 게시판으로 이동합니다.

1. 열린 Chrome에서 직접 Daum에 로그인합니다.
2. 대상 게시판 접근이 완료되어 게시글 목록이 보이는지 확인합니다.
3. 콘솔 창으로 돌아와 Enter 키를 누릅니다.
4. 수집할 시작 페이지와 종료 페이지를 입력합니다.
5. 프로그램이 지정한 페이지 범위 안에서 `Re:`로 시작하는 글만 열람합니다.
6. 지정한 종료 페이지까지 1페이지씩 순서대로 이동하면서 수집합니다.
7. 수집 완료 후 아래 파일이 생성됩니다.

```text
학교회계디딤돌_묻고답하기_Re글_수집.docx
```

수집량이 많아 한 파일이 예상 300페이지를 넘을 경우에는 아래처럼 순번이 붙은 워드파일로 나뉘어 저장됩니다.

```text
학교회계디딤돌_묻고답하기_Re글_수집_001.docx
학교회계디딤돌_묻고답하기_Re글_수집_002.docx
```

중간 오류가 발생하거나 사용자가 중단하면 현재까지 수집한 내용이 아래 파일로 저장됩니다.

```text
autosave.docx
```

분할 저장 중에는 현재 작성 중인 파일이 `autosave.docx`와 `autosave_002.docx` 같은 순번 autosave 파일로도 저장됩니다.

`autosave.docx`는 중간 오류에 대비한 백업 파일입니다. 정상 완료 결과는 `학교회계디딤돌_묻고답하기_Re글_수집.docx` 또는 순번이 붙은 결과 파일을 사용하면 됩니다.

페이지 이동은 1 → 2 → 3 순서로 진행합니다. Daum Cafe의 `다음` 화살표는 경우에 따라 10페이지 같은 다음 묶음으로 이동할 수 있어서, 프로그램은 화살표를 누르지 않고 `현재 페이지 + 1` 값을 직접 요청합니다.

실행 중 페이지 범위 입력 예:

```text
시작 페이지 (Enter=1): 10
종료 페이지 (Enter=1480): 20
수집 범위: 10 ~ 20 페이지
```

위처럼 입력하면 10페이지로 먼저 이동한 뒤 10, 11, 12 ... 20페이지만 수집합니다.

시작 페이지를 비워두면 현재 페이지부터 시작합니다. 종료 페이지를 비워두면 감지된 마지막 페이지까지 수집합니다.

## 수집 기준

목록에서 제목 앞에 들여쓰기, 공백, `ㄴ`, 답글 표시 아이콘 텍스트 등이 있더라도 이를 제거한 뒤 실제 제목이 `Re:`로 시작하는지 판단합니다.

예:

- `ㄴ Re: 2인이상수의 공고시 문의사항있습니다.` → 수집
- `수의계약 견적 공고시 문의사항` → 제외

워드파일에는 각 게시글의 아래 2가지 항목만 저장합니다.

- 제목
- 본문

댓글, 이전 글 인용문, 메뉴명, 카페 안내문, 광고, 목록 텍스트, 추천/조회수 등은 최대한 제외하도록 구현되어 있습니다.

## 디버그 파일

제목 또는 본문 추출에 실패하면 `debug` 폴더에 자료가 저장됩니다.

- `.png`: 실패 당시 화면 스크린샷
- `.html`: 실패 당시 page source

Daum Cafe의 CSS 선택자가 바뀌었거나, 특정 글의 HTML 구조가 다른 경우 이 파일을 확인해 `main.py`의 선택자 목록을 조정할 수 있습니다.

## 자주 생기는 문제

### ChromeDriver 다운로드 실패

회사/학교망에서 다운로드가 막혀 있을 수 있습니다. 네트워크를 바꾸거나 보안 정책을 확인한 뒤 다시 실행하세요.

```powershell
python main.py
```

### 로그인했는데도 수집이 시작되지 않음

콘솔에서 Enter를 누르기 전에 게시판 목록이 실제로 보이는 상태인지 확인하세요. 카페 가입 승인, 등급 제한, 게시판 권한 제한이 있으면 프로그램은 우회하지 않습니다.

프로그램이 아래와 같은 문구를 감지하면 실제 글로 저장하지 않고 다시 기다립니다.

```text
회원님은 아직 로그인을 하지 않으셨어요.
묻고답하기 게시판 권한 안내
읽기권한
```

이 경우 열린 Chrome 창에서 직접 로그인한 뒤, 게시판 목록이 보이는 상태까지 이동하고 콘솔에서 Enter를 누르세요.

### 본문이 비어 있거나 엉뚱한 내용이 저장됨

`debug` 폴더의 `.html` 파일을 확인해야 합니다. Daum Cafe 화면 구조가 다르면 `ARTICLE_TITLE_SELECTORS`, `ARTICLE_BODY_SELECTORS`, `REMOVE_IN_BODY_SELECTORS` 값을 조정해야 할 수 있습니다.

### autosave.docx Permission denied 오류

`autosave.docx` 또는 결과 `.docx` 파일이 Word에서 열려 있으면 Windows가 파일을 잠가 저장이 실패할 수 있습니다. 가능하면 열려 있는 워드파일을 닫고 다시 실행하세요.

프로그램은 저장 권한 오류가 나면 중단하지 않고 아래처럼 시간이 붙은 대체 파일로 저장합니다.

```text
autosave_20260428_115201.docx
학교회계디딤돌_묻고답하기_Re글_수집_20260428_115201.docx
```

## 결과 문서 형식

문서 제목:

```text
학교회계 디딤돌 묻고답하기 Re글 수집
```

각 게시글은 다음 형식으로 들어갑니다.

```text
제목: Heading 1
본문: 일반 문단
게시글 사이: 강제 페이지 나눔 없음. 바로 이어서 다음 글 저장
```

## 300페이지 분할 기준

`python-docx`는 Microsoft Word가 문서를 실제로 배치한 뒤의 정확한 페이지 수를 실행 중에 알 수 없습니다. 그래서 이 프로그램은 제목과 본문의 글자 수, 줄 수를 기준으로 페이지 수를 예상하고, 현재 파일이 예상 300페이지를 넘을 것으로 보이면 새 워드파일을 시작합니다.

기준값은 `main.py` 상단에서 조정할 수 있습니다.

```python
MAX_PAGES_PER_FILE = 300
APPROX_CHARS_PER_LINE = 46
APPROX_LINES_PER_PAGE = 36
```
